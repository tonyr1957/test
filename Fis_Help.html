Introduction
 Fuel Information System is a tool built to help end users in automating the ongoing process of downloading the files from State Transportation Department, processing the data and uploading to STARS. This document was made to help the users use the application effectively.

Using the Fuel Information System
Page Name: FisHome.jsp

This is the home/default page for FIS Application.

The functionality on this page includes:

Do Gascard FTP– Click on this Button to start downloading GC Transaction File from transportation department and import data into FIS database. Also it triggers messages to the users about the status of the Transfer and action needs to be taken.

Do Wex FTP- Click on this Button to start downloading WEX Transaction File from transportation department and import data into FIS database. Also it triggers messages to the users about the status of the Transfer and action needs to be taken.

Initial Processing– Clicking on this button starts the initial processing like calculating transaction amount, loading Charge PCAs for the vehicles and assigning expense SO based on product codes.

Process Charges– After initial processing is done, click this button to verify the batch Total (Sum of Transaction amount taken from the database) against the net statement charges (entered in the Net Statement Charges field of the FisHome page

Create Upload File– Click this button to Create STARS batch upload file. Once clicked messages are triggered to the users about the status of the Transfer and the actions needed to be taken.

Reset Process-Click on this button to reset the above process and start all over again.

Upload to Stars– Clicking on this button will start the FTP Process of uploading th created batch file to STARS system. Once clicked messages are triggered to the users about the status of the Transfer and the actions needed to be taken.

Reports
Page Name: Reports.jsp
This page helps to view different reports of FIS Application.

The functionality on this page includes:

All Transaction Reports– This will give a scrollable report of records in transaction table between the Posting Dates entered in the screen for a specific vendor. Click on the edit button for updating a specific transaction record. This will take you to the transaction maintenance page.

Duplicate Transaction Reports- This will give a Duplicate Records Scrollable Report of records in transaction table between the Posting Dates entered in the screen. Clicking on the duplicate count will take you to the duplicates page where you can able mark a record as duplicate. Refer Duplicate page description for more details.

Balance Report- This will give all Batch Total/Credit Exceed Charges Report for all the batches run between the entered posting dates on the screen.

Charge PCA/Expense SO Null Report– This will generate a report of Charge PCA and Expense SO null values in the transaction table for those records that are not been processed for Batch Upload.

Equipment PCA Maintenance
Page Name: EquipPCA.jsp
This page is used to maintain the Equipment PCA table.


The functionality of this page includes:

Find:
 This function allows searching the database for a specific property number or property number, effective charge date combination.

Add:
 This function allows adding a new property, effective charge date combination to the database. Property Number and Effective charge date fields are mandatory entry.

Update:
 This function allows updating an existing property, effective charge date combination to the database. Property Number and Effective charge date fields are mandatory entry. This button will show up in case of editing an existing property number.

For editing and updating an existing property number, do a search on the property number (Find PCA) and then click on the edit icon/delete icon for the record to be updated/deleted respectively.

ExpenseSO/PCA Null Report
Page Name: NullReport.jsp

This page is used to generate report of Expense SO/PCA Null value records in transaction table from those records that are not processed for Upload to STARS (The records are said to be processed when the creation of batch file completes). This page will automatically pop up if there are any null values in the PCA/Expense SO at the end of initial processing. You can also get into this page by clicking the link on the FisHome page or Reports page.

If the reports shows some records, it indicates those record have null values either in PCA column or Expense SO Column.

Null in Charge PCA is caused because of no matching records in Equipment PCA table. New vehicle might be present in the transaction, which might not be added in Equipment PCA table or the vehicle might be out of use but the old card could have been used for transaction. Check with the purchasing department and do the necessary action. Click on the vehicle for the record to go to the Equipment PCA Maintenance screen.

Null in Expense SO indicates that there is no valid product code for the transaction to look for the Expense SO value. Click on the Product Code to go to the Product Expense SO maintenance Screen and do the necessary.
